在负责过江雪天演舆情项目后，我们发现辅助和周边服务存在较大的不足。因此，在天演舆情中，除了数据、存储、分析和研判等方面的对比外，几乎无法与在资金、人际关系和先发优势等方面具备优势的前辈个体有所差异。

为此，天演将专注于构建和完善计算资源等周边服务。在这一领域，幸运的是，很少有人涉足，而且技术难度相当大。这需要更多的时间以及更多创新性的思考和方法，以规避计算资源支出方面的问题。

更令人振奋的是，这些方面与江雪天演项目的性质高度契合。通过与周边应用的对接，可以使天演项目在原本已经很优秀的基础上更具特色，并提升相关领域的门槛。

有趣的是，在我将 "江雪虹霓实验室计算平台" 发布到 Behance 不久后，我收到了一个关于 [Vantage](https://www.vantagemarkets.com/en/) 3D 设计师助理职位的邀请。

和他们的设计师和主管聊了很久，感觉他们对设计的追求和开发的落地效果都有较高的标准。并不是像我一样构思玩设计之后，还需要进一步设计动效、交互等方面的细节。之后还需分割成一个一个的组件方便以后会用得到，有时候真觉得。

如果一个人只靠设计，或许真的可以完成一个项目的前期运营和开发，而且还很快。目前流行的 [Webflow](https://webflow.com/) 或 [WIX](https://www.wix.com/) (ps: 不要问我怎么知道的，之前看朋友在用 wix ），通常是 前者作为 CMS，之后 Figma 设计、[Lottie lab](https://www.lottielab.com/) 或者 [Lottiefiles](https://lottiefiles.com/)  作为动效设计。

像是 Vantage 的设计就用到了 [spline](https://spline.design/)，我承认他是一个好的在线 3D 设计平台(如果不卡的话，那就更好了)；我有注意到沈毅也在做 Web 3D 设计的平台，但性能方面还是受到了 WebGPU 方面的限制。如果你有兴趣被折磨，可以尝试下 WebGPU 的光线追踪 [Demo](https://gnikoloff.github.io/webgpu-raytracer/)

总之像这些新的基于 HTML5 Canvas 2D 渲染引擎确实为设计领域增加了不少色彩（我已经不知道多久没有打开我的 Photoshop 了）

::page-fold
真的很馋鹅厂的设计和技术，动效、3D、特效、原画真的好看又华丽。将来如果江雪也能拥有，就更好了。
::

但这些都不是重点，重点是：

现在。江雪广泛使用由 Three.js 和 GSAP.js 构建的效果。而前的琼楼仅仅是一些基础常用需求的组件，因此在未来可能更加接近于 PAG 的引入和设计流程，从而实现与琼楼的无缝衔接，以缩短动画效果的构思时间和场景构建。

虽然动效、3D、特效、原画这些江雪都没有或者有很大的缺陷，但未来也许会有。
